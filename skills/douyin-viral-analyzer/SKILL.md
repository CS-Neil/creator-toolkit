# 爆款文案拆解器

> 输入一份 markdown 文案 → AI 深度拆解分析 → 写入飞书多维表格。

## 前置条件

- 飞书多维表格已创建：[爆款文案拆解库](https://jcnw6e2ixp8h.feishu.cn/base/REfRboMTdaJEX6s6o3EcmQqxnBb)
- 用户提供一份 `.md` 格式的文案文件（路径或直接粘贴内容均可）

## 快速开始

用户输入文案文件路径即可，例如：

> 分析这份文案 /Users/chenhui/Downloads/7661857348968860980.md

也支持直接粘贴文案内容：

> 分析这段文案：<粘贴 markdown 内容>

## 全流程步骤

### Step 1: 读取文案

如果用户提供了文件路径，使用 Read 工具读取：

```
Read <用户提供的文件路径>
```

如果用户直接粘贴了文案内容，直接进入下一步。

从文案中提取以下基本信息（如果原文中存在）：
- **标题**：通常在第一行 `# 标题` 或文件名中
- **作者/账号**：从标题、正文或用户补充中获取
- **平台**：从文案风格推断（抖音/公众号/小红书/B站/微博/知乎/视频号/其他）
- **内容形式**：从文案特征推断（短视频/长视频/图文/纯文字/图集）
- **原始链接**：如果用户提供了链接
- **发布日期**：如果能从文案中获取

### Step 2: AI 深度拆解分析

**重要：此步骤由 AI 直接完成，无需调用外部工具。**

按照以下 7 大维度对文案进行**深度分析**，每个字段都要给出具体、有洞察的结论，而不是泛泛而谈：

#### 2.1 选题与定位
- **选题类型**（多选，1-3个）：热点借势 / 情感共鸣 / 认知反差 / 实用干货 / 争议观点 / 其他
- **目标人群**：打动的是什么人？他们的兴趣、焦虑、渴望是什么？
- **痛点切入**：解决了什么具体问题？还是放大了什么情绪？为什么这个痛点是有效的？
- **核心观点**：一句话概括，这篇文章到底想说什么？

#### 2.2 标题与钩子
- **标题结构**（多选，1-2个）：数字型 / 悬念型 / 对比型 / 恐吓型 / 承诺型 / 故事型 / 提问型 / 其他
- **情绪关键词**：标题/开头用了哪些高唤醒词？逐个列出并说明为什么有效
- **好奇心缺口**：标题/开头制造了什么信息差让人不得不点/读下去？

#### 2.3 结构与节奏
- **开头钩子**：前3句/前3秒怎么抓注意力？用了什么具体手法？（冲突/悬念/共鸣/反常识/现状冲击）
- **情绪曲线**：用箭头描述整篇文案的情绪起伏节奏（如：冲击→低谷→爬升→高潮→回落），每个阶段标注对应内容
- **信息密度**（单选）：高 / 中 / 低 —— 给出判断依据
- **结尾设计**：怎么收尾的？总结升华 / 反转 / 互动引导 / 行动号召？具体怎么做的？

#### 2.4 内容核心
- **论证方式**（多选，1-3个）：讲故事 / 列数据 / 举例 / 引用权威 / 个人经历 / 其他
- **金句摘录**：逐条列出可截图转发的句子（通常3-5条），每条单独一行
- **情绪价值**（多选，1-2个）：让人爽 / 让人焦虑 / 让人感动 / 让人愤怒 / 实用干货 / 其他
- **认知增量**：读者看完后获得了什么新知或新视角？具体是什么？

#### 2.5 互动与传播
- **传播密码**：为什么有人愿意转发？身份表达 / 社交货币 / 情绪共鸣？具体分析
- **评论亮点**：预判评论区高赞评论会说什么？为什么这些评论会成为高赞？

#### 2.6 学习与复用（重点）
- **可复用结构**：拆出完整的内容框架，用层级缩进结构展示：
  ```
  【Hook层】xxx
  【铺垫层】xxx
  【主体层】xxx
    ├── 要点1
    ├── 要点2
    └── 要点3
  【升华层】xxx
  【结尾层】xxx
  ```
- **仿写思路**：如何用自己的内容替换原文？给出至少2个具体选题建议
- **我的启发**：列出 3-5 条核心启发，每条一句话，要具体可执行
- **评分**（单选）：⭐ / ⭐⭐ / ⭐⭐⭐ / ⭐⭐⭐⭐ / ⭐⭐⭐⭐⭐
- **标签**（多选，1-3个）：待仿写 / 已仿写 / 经典案例 / 可复用结构 / 垂直领域标杆

#### 2.7 互动数据
- 点赞数、评论数、收藏数、转发数 —— 如果用户提供了数据则填写，否则留空不写入

分析完成后，调用 `lark-cli base +record-upsert` 写入：

```bash
lark-cli base +record-upsert \

    "文案标题": "<标题>",
    "作者/账号": "<作者>",
    "平台": "<平台>",
    "内容形式": "<内容形式>",
    "链接": "<原始链接，没有则不填>",
    "发布日期": "<发布日期 YYYY-MM-DD，没有则不填>",
    "拆解日期": "<今日日期 YYYY-MM-DD>",
    "选题类型": ["<分析结果>"],
    "目标人群": "<分析结果>",
    "痛点切入": "<分析结果>",
    "核心观点": "<分析结果>",
    "标题结构": ["<分析结果>"],
    "情绪关键词": "<分析结果>",
    "好奇心缺口": "<分析结果>",
    "开头钩子": "<分析结果>",
    "情绪曲线": "<分析结果>",
    "信息密度": "<分析结果>",
    "结尾设计": "<分析结果>",
    "论证方式": ["<分析结果>"],
    "金句摘录": "<分析结果>",
    "情绪价值": ["<分析结果>"],
    "认知增量": "<分析结果>",
    "传播密码": "<分析结果>",
    "评论亮点": "<分析结果>",
    "可复用结构": "<分析结果>",
    "仿写思路": "<分析结果>",
    "我的启发": "<分析结果>",
    "评分": "<分析结果>",
    "标签": ["<分析结果>"]
  }' --as user
```

**字段写入规则**：
- 文本字段：直接传字符串，换行用 `\n`
- 单选字段：传字符串，如 `"高"`、`"抖音"`、`"⭐⭐⭐⭐⭐"`
- 多选字段：传字符串数组，如 `["认知反差", "实用干货"]`
- 数字字段：传 JSON 数字，如 `12345`
- 日期字段：传 `"YYYY-MM-DD"` 格式字符串
- 链接字段：传 URL 字符串，如 `"https://www.douyin.com/video/xxx"`
- **不要写入没有值的字段**（如没有互动数据就不要传点赞数等字段）

## 常见问题

### 飞书写入失败

1. 检查认证状态：`lark-cli auth whoami --as user`
2. 如果未登录：`lark-cli auth login --as user`
3. 检查 Base 权限：确认用户有编辑权限

### 分析质量提升

1. 仔细阅读完整文案，不要只看开头就下结论
2. 重点关注「可复用结构」和「开头钩子」——这是仿写的核心
3. 金句摘录要精准，只摘真正有传播力的句子，每句单独一行
4. 情绪曲线要描述具体的高低起伏，标注每个阶段对应的内容段落
5. 仿写思路要给出具体、可操作的选题建议，避免空泛
